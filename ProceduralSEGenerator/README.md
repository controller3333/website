# プロシージャルSEジェネレーター

ノードベースのプロシージャル効果音合成ツール（GameSynth ライク）。ブラウザだけで動く、依存ゼロのバニラ JS アプリです。

基本となる音を出すノード・加工するノード・組み合わせるノードを線で繋いで、最終的な効果音（ワンショット SE）を作ります。各ノードはオフラインでサンプルバッファ（`Float32Array`）を処理し、グラフ全体を評価して 1 本の音をレンダリングします。再生・WAV 書き出し・波形表示に対応。

## 起動

依存ゼロの Node サーバ `server.js` で起動します（設定の保存に使うため推奨）。

```sh
cd ProceduralSEGenerator
node server.js 5599
# → http://localhost:5599
```

静的配信だけでも動きます（`python -m http.server 5599` 等）。ただし設定の**ファイル保存**は `node server.js` のときのみ有効で、それ以外では localStorage にフォールバックします。

### 設定の保存

「日本語」表示などの設定は、`server.js` 経由なら**アプリフォルダ内の `settings.json`** に保存されます。フォルダごと別PCへコピーすれば設定もそのまま引き継がれます（`GET/POST /api/settings`）。サーバが無い場合はブラウザの localStorage に保存します。

## 使い方

- 左パレットのノード名を**クリック**するとキャンバス中央に追加されます。
- ノードの**出力ポート（右の●）から入力ポート（左の●）へドラッグ**して接続。配線を**クリックすると削除**。入力ポートからドラッグすると既存の接続を掴み直せます。
- ノードのヘッダをドラッグで移動。`×` で削除（Output は固定）。選択中ノードは `Delete` キーで削除。
- **コピー＆ペースト**: 選択ノードを `Ctrl+C`→`Ctrl+V`（カーソル位置に貼付）、`Ctrl+D` で複製。
- **ノートノード**: パレットの「Note」を追加してメモを記入。ノードの上端へドラッグすると連結し、そのノードを動かすとノートも追従（⛓ ボタンか引き離しで解除）。
- **日本語表示**: 既定でオンです。上部メニューの「日本語」を切り替えると、ノード名・カテゴリ名・パラメータ名・選択肢が切り替わります。
- **パレットのツールチップ**: 左のノード一覧にマウスを乗せると、そのノードの説明が表示されます。
- **ノード試聴**: ノードにカーソルを重ねて `Ctrl+Space`。そのノードまでをレンダリングして**その地点の音**を再生します（信号の途中経過を確認できる）。制御信号（Sweep/LFO/Envelope 等で値が±1を超えるもの）は安全のため自動で音量を下げて再生します。
- 背景（空白）ドラッグで**パン**、**ホイールでズーム**。**空白を右クリック**するとコンテキストメニューが開き、カテゴリ→ノードを選んでその位置に挿入できます。
- ノードの各**パラメータにマウスを乗せる**と、その説明がツールチップで表示されます（日本語トグルに連動）。
- 上部バー: `再生`（または `Space`）/`停止`、`長さ`（秒）、`レート`（サンプルレート）、`ループ`、`自動`（パラメータ変更で自動再レンダリング）、`Pitch±`（再生ごとのランダムなピッチ揺らぎ cent）、`🎲`（揺らして再生）、`生成`、`WAV`（書き出し）。
- 右上: プリセット選択、グラフの `保存`/`開く`（JSON）、`消去`。
- 最終的に音が出るのは **Output** ノードに繋がった信号だけです。

### ステレオについて

内部処理はモノのサンプルバッファですが、**Panner** がモノ→ステレオの境界になります。Panner より下流（Mixer / Output）はステレオを扱い、ステレオ非対応のノードにステレオ信号が入った場合はグラフ評価時に自動でモノへダウンミックスします。ステレオ信号は WAV も 2ch で書き出されます。

### ピッチ揺らぎ（複数同時発音のランダム化）

- **Oscillator の Unison/Detune**: 1つのオシレータ内でデチューンした最大7声を同時に鳴らし、厚み・うなりを作ります。
- **Pitch±（cent）＋ 🎲**: 再生のたびに ±cent の範囲でランダムにピッチを変え、ノイズの seed も変えます。足音や着弾音など「繰り返しても機械的に聞こえない」SE 作りに有効。`生成` は直前の揺らぎ値を保持（同じ音を再現）、`🎲` は新しい揺らぎで再生します。

## 仕組みのメモ

- 接続は全て **モノラルのサンプルバッファ**（長さ = `duration × sampleRate`）を運びます。音声信号も制御信号（CV/エンベロープ/Sweep）も同じ型で、繋ぎ方で意味が決まります。
- レンダリングは Output から入力を再帰的に pull し、各ノードの出力をレンダリング毎にメモ化。グラフに循環があっても検出して無音で打ち切ります。
- フィルタは RBJ Cookbook の biquad を 1 サンプルずつ処理（CV があればサンプル単位でカットオフ再計算）。
